[사용자 매뉴얼]

1. 개요
이 시스템은 사내 문서, 매뉴얼, FAQ를 기반으로 질문에 답변하는 문서 검색형 챗봇입니다.
모든 답변은 사전에 제공된 문서를 근거로 생성됩니다.

2. 시스템 구성
- LLM: Ollama (llama3.1 모델)
- 임베딩 모델: nomic-embed-text
- 벡터 DB: FAISS (로컬)
- 프레임워크: LangChain

3. 사용 방법
1) data 폴더에 문서를 저장합니다.
2) 챗봇 애플리케이션을 실행합니다.
3) 질문을 입력하면 관련 문서를 검색하여 답변합니다.
4) 종료하려면 exit 또는 quit을 입력합니다.

4. 문서 작성 가이드
- 한 문단에는 하나의 주제만 작성하세요.
- 표보다는 자연어 문장이 검색에 유리합니다.
- 너무 긴 문장은 적절히 나누는 것이 좋습니다.

5. 주의 사항
- 문서에 없는 내용은 답변되지 않습니다.
- 개인 정보나 민감한 정보는 문서에 포함하지 마세요.
- 문서를 수정한 경우 벡터 인덱스를 다시 생성해야 합니다.

6. 벡터 인덱스 재생성 방법
- faiss_index 폴더를 삭제합니다.
- 챗봇을 다시 실행하면 자동으로 인덱스가 재생성됩니다.

7. 오류 대응
- Ollama 서버가 실행 중인지 확인하세요.
- 모델이 설치되어 있는지 확인하세요.
- 문서 인코딩은 UTF-8을 권장합니다.

8. 확장 가능 기능
- PDF 문서 로딩
- 대화 메모리 추가
- 웹 UI 연동 (FastAPI, React 등)
- 사용자 권한별 문서 접근 제어
